Frontend — React App

A modern SPA built with React 18 and TypeScript. Uses Redux Toolkit for global state management, React Router for client-side routing, and Material-UI for a polished component library.

React 18TypeScriptRedux Toolkit MUIReact RouterVite
⚡

Backend — Express API

A robust RESTful API server built with Node.js and Express.js, following the MVC pattern. Handles authentication with JWT and bcrypt, request validation, file uploads, and Stripe webhooks.

Node.js 18+Express.jsJWT BcryptMulterStripe SDK
🍃

Database — MongoDB + Redis

MongoDB Atlas is the primary database with Mongoose ODM for schema definition and validation. Redis handles session caching, rate limiting, and transient data to boost performance.

✨ Platform Features

Everything You Need in an LMS

From course creation to payment and certification — the platform covers the full lifecycle of online education.

👤

User Management

Student, Instructor & Admin roles with JWT-secured authentication, password hashing, and profile management.

📚

Course Management

Full CRUD for courses with modules, lessons, video content, attachments, and publish/unpublish workflows.

📝

Quiz & Assessments

Timed quizzes with multiple-choice questions, automatic grading, and detailed result pages with score history.

💳

Payment Processing

Stripe integration for secure course purchases with checkout sessions, success/failure pages, and webhook handling.

🎓

Certificate Generation

Automatic certificate issuance upon 100% course completion, visible from the student dashboard.

🔔

Real-time Notifications

In-app notifications for enrollment, course updates, quiz results, and important platform events.

📊

Analytics Dashboards

Rich analytics for instructors (revenue, enrollments) and admins (platform-wide KPIs and user stats).

🎯

Progress Tracking

Students can track lesson completion, overall course progress percentage, and quiz scores in one dashboard.

🗄️ Data Models

MongoDB Schema Design

Mongoose models with strict typing, validation, and relational references across the platform.

User
  • _id: ObjectId
  • name: String
  • email: String (unique)
  • password: String (hashed)
  • role: "student"|"instructor"|"admin"
  • isApproved: Boolean
  • createdAt: Date
Course
  • _id: ObjectId
  • title: String
  • instructor: ref: User
  • modules: [ Module ]
  • price: Number
  • category: String
  • isPublished: Boolean
Enrollment
  • _id: ObjectId
  • student: ref: User
  • course: ref: Course
  • progress: Number (0–100)
  • completedLessons: [ ObjectId ]
  • certificateIssued: Boolean
  • enrolledAt: Date
Quiz
  • _id: ObjectId
  • course: ref: Course
  • title: String
  • questions: [ Question ]
  • timeLimit: Number (minutes)
  • passingScore: Number
Payment
  • _id: ObjectId
  • student: ref: User
  • course: ref: Course
  • amount: Number
  • stripeSessionId: String
  • status: "pending"|"success"|"failed"
  • paidAt: Date
Notification
  • _id: ObjectId
  • user: ref: User
  • type: String (enum)
  • message: String
  • isRead: Boolean
  • createdAt: Date
